## 一、项目概览

一个温暖配色的 Todo-List 桌面应用。支持三类任务管理（DDL / 每日 / 每周），含截止日期、超时提醒、系统托盘、开机自启、单文件打包。

├─────────────────────────────────────────────┤

## 三、架构设计

### 分层架构

```
resources/style.qss          ← 纯视觉层（QSS 样式表）
todo_app/ui/                 ← UI 层（Qt Widgets）
todo_app/ui/task_table_model ← 数据桥梁（QAbstractTableModel）
todo_app/database.py         ← 数据层（SQLite CRUD）
todo_app/models.py           ← 数据结构（dataclass）
todo_app/utils.py            ← 工具层（路径/注册表）
```

### 数据流

```
用户操作 → 信号 → MainWindow slot → DatabaseManager CRUD → SQLite
                                         ↓
                              TaskTableModel.refresh_data()
                               → beginResetModel()
                               → get_all_tasks()
                               → endResetModel()
                               → QTableView 自动重绘
```

### 关键设计决策

| 决策 | 说明 |
|------|------|
| 不用 QSortFilterProxyModel | 过滤在 SQL 层（WHERE），排序在 Python 层（list.sort），更可控 |
| `beginResetModel/endResetModel` | 替代 `layoutChanged.emit()`，防止无障碍层崩溃 |
| `task_type` 字段 | DDL/Daily/Weekly 存同一张表，用字段区分，按标签过滤 |
| 委托实现行内按钮 | `QStyledItemDelegate` 绘制 ✕ 按钮，`editorEvent` 处理点击 |
| HKCU 注册表 | 开机自启用 `HKCU\...\Run`，无需管理员权限 |
| 资源路径双模式 | `sys.frozen` 判断开发 vs 打包，`sys._MEIPASS` 处理 PyInstaller 临时目录 |

## 六、关键技术点

### 1. QAbstractTableModel 的正确用法

```python
class TaskTableModel(QAbstractTableModel):
    def refresh_data(self, filter_params=None):
        self.beginResetModel()          # ← 通知：我要改了，别读
        self._tasks = self._db.get_all_tasks(self._filter)
        self.endResetModel()            # ← 通知：改完了，可以读了

    def data(self, index, role):
        if role == Qt.DisplayRole: ...   # 显示文字
        if role == Qt.ForegroundRole: ... # 文字颜色
        if role == Qt.TextAlignmentRole: ... # 对齐
```

**为什么用 `beginResetModel/endResetModel`**：直接 `emit(layoutChanged)` 会导致 Qt 无障碍层在数据变更中途查询已删除的索引，产生 `QAccessibleTable::child: Invalid index` 错误。

### 2. 自定义 Delegate 实现行内按钮

```python
class DeleteButtonDelegate(QStyledItemDelegate):
    delete_clicked = pyqtSignal(int)   # 发出 task_id

    def paint(self, painter, option, index):
        # 在指定 cell 绘制 ✕ 按钮
        btn_rect = QRectF(x, y, 28, 22)
        painter.drawRoundedRect(btn_rect, 4, 4)
        painter.drawText(btn_rect, AlignCenter, "✕")

    def editorEvent(self, event, model, option, index):
        # 点击检测：判断鼠标是否在按钮区域
        if event.type() == MouseButtonRelease:
            if btn_rect.contains(event.pos()):
                task = model.get_task(index.row())
                self.delete_clicked.emit(task.id)
```

### 3. 动态列布局切换

DDL 和 Daily/Weekly 的列数和宽度不同。切换标签时：

```python
def _apply_column_layout(self, task_type):
    for c in range(10):
        self._table.setItemDelegateForColumn(c, None)  # 先清除所有委托
    if task_type == "ddl":
        # 5 列：标题(Stretch) | 优先级(64) | 截止(Stretch) | 状态(72) | 操作(44)
        action_col = 4
    else:
        # 3 列：标题(Stretch) | 优先级(72) | 操作(44)
        action_col = 2
    self._table.setItemDelegateForColumn(action_col, self._delete_delegate)
```

**注意**：每次切换必须先清除所有列的委托，否则旧列索引的委托会残留在新布局的其他列上。

### 4. 超时判断（字符串比较）

```python
def _is_overdue(self, task):
    if not task.due_date or task.status != "pending":
        return False
    return task.due_date < datetime.now().strftime("%Y-%m-%d %H:%M")
```

因为日期格式统一为 ISO 字符串（`"2026-05-27 14:30"`），直接用字符串比较即可，无需解析为 datetime 对象。旧任务仅有日期（`"2026-05-27"`），字符串比较同样有效。

## 八、运行与打包

```bash
# 开发运行
python main.py

# 打包
pyinstaller --noconsole --onefile --name "TodoList" \
    --add-data "resources/style.qss;resources" \
    --add-data "resources/app_icon2.png;resources" \
    main.py

# 或使用 spec 文件
pyinstaller TodoList.spec
```

输出：`dist/TodoList.exe`（~34MB，单文件，无控制台窗口）
